ztsc

A fast, memory-lean TypeScript type checker, written in Zig.

No JS emit, no Node runtime — a single static binary with zero dependencies, nothing but the Zig standard library. It checks a large, well-defined subset of strict-mode TypeScript with diagnostics that match the TypeScript compiler, byte-identical at any parallelism.

benchmarks

Real packages, identical inputs

Each package's published .d.ts is vendored at a pinned version and both tools check identical inputs, each tool at its own default checker count (ztsc 8, tsgo 4), on an Apple M4. Wall clock is the median of 11 runs timed with a monotonic nanosecond timer around the whole process — so small-package ratios are real, not timer rounding; peak RSS is the median of 5 runs under /usr/bin/time -l. One warm-up per configuration.

getting started

Build it, point it at a project

v0.0.1 is not on npm yet (once it is: bunx ztsc). Until then, build from source — all you need is Zig 0.16.0; there are no dependencies to fetch:

git clone https://github.com/gustavoschmidt/ztsc
cd ztsc
zig build -Doptimize=ReleaseFast   # -> zig-out/bin/ztsc

Point it at a project and it does the rest:

ztsc                       # finds tsconfig.json in cwd or a parent
ztsc -p path/to/project    # explicit tsconfig
ztsc src/main.ts           # or explicit entry files

Diagnostics are tsc-style, with source excerpts and matching error codes:

demo.ts:1:19 - error TS2322: Type 'string' is not assignable to type 'number'.

1 const n: number = "hi";
                    ~~~~

Pre-release. ztsc is not ready for production use. It checks a large, well-defined subset of strict-mode TypeScript — watch mode and an LSP are the biggest gaps — and unsupported syntax produces a clear "not supported" diagnostic, never a wrong answer or a crash. The full list is on the usage page.
